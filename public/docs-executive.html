Infraestructura &
Flujo de Datos

01

Procesamiento de Audio Multimodal

La arquitectura permite cargar audios, extraer duración real, detectar silencios operativos, conservar la transcripción y evaluar cumplimiento con Gemini 2.5 Flash bajo criterios configurados por campaña y ficha de calidad.

02

Sistema de Colas Asíncronas

Utilizamos Laravel Jobs con Redis y workers separados para transcripción, scoring IA y tareas operativas. Esto evita bloquear la interfaz cuando varios monitores cargan audios en paralelo.

03

Almacenamiento de Datos Robusto

Cada interacción, evaluación, respuesta de feedback, disputa, auditoría y señal de voz queda persistida en PostgreSQL, segmentada por campaña, asesor, supervisor, monitor y estado de revisión.

04

Jerarquía y Segmentación de Campañas

El sistema implementa una jerarquía rigurosa para la asignación de usuarios y visualización de datos. Esto garantiza que agentes y supervisores (por ejemplo, del área de Prepago o Upgrade) interactúen únicamente con su respectivo ámbito operativo, evitando cruces de información.

El Corazón de la Precisión

El motor de IA trabaja sobre criterios de calidad configurados, señales operativas, transcripción, tono conversacional, tiempos muertos y evidencia capturada en cada interacción.

CT

Contexto Operativo

Cada evaluación incorpora ficha, campaña, criterios críticos, contexto operativo y evidencia del audio para que la IA evalúe contra reglas reales del negocio.

DT

Control de Salida

El sistema exige JSON estructurado, valida resultados y registra errores del proveedor para diferenciar límites, facturación, configuración y fallas temporales.

GR

Feedback Accionable

Los resultados alimentan seguimiento del asesor, aceptación de feedback, disputas, alertas e insights ejecutivos para operaciones y cliente.

Ingeniería de Prompts y Lógica de Decisión

La IA no se limita a comparar palabras. La evaluación combina criterios de ficha, resumen conversacional, comportamiento del asesor, respuesta del cliente, tiempos muertos y señales de experiencia para producir una lectura revisable por monitores.

"El objetivo no es reemplazar al monitor: es entregarle evidencia, priorización y una primera lectura consistente para que la revisión humana sea más rápida y defendible."
  • Indicadores de voz: Se exponen señales como ritmo, velocidad, pausas, tiempos muertos y riesgo de experiencia para apoyar feedback operativo.
  • Scoring y Localización: La nota se calcula con los pesos y criterios definidos en cada formulario de calidad, mapeados en español ("Cumple", "No Cumple", "No Aplica", "No Encontrado") tanto en el portal web como en la APK móvil.
  • Fallas críticas: Los criterios marcados como críticos impactan la calificación y disparan alertas para revisión prioritaria.

Arquitectura Técnica &
Documentación de Ingeniería

01 Instalación y Despliegue

El sistema está diseñado para entornos de contenedores (Docker) y servidores tradicionales Linux. Sigue un flujo estándar de Laravel con optimizaciones para alta disponibilidad.

Requisitos del Servidor

  • PHP Runtime 8.3+
  • Base de Datos PostgreSQL 15+
  • Key-Value Store Redis 7+
  • Servidor Web Nginx / Apache

Comandos de Inicialización

# Instalar dependencias
composer install --optimize-autoloader
npm ci && npm run build

# Migrar y Sembrar
php artisan migrate --force
php artisan db:seed --class=RoleSeeder

# Iniciar Workers
php artisan queue:work redis --tries=3

Seguridad y Gestión de Accesos

Implementamos un modelo RBAC (Role-Based Access Control) estricto. Los datos sensibles están protegidos mediante el cifrado nativo de Laravel (AES-256) y las llaves de API se gestionan de forma cifrada en la base de datos.

Fórmula de Usuarios Autogenerados: Para garantizar consistencia, los nombres de usuario se crean de forma automatizada mediante la regla: 1ra letra del nombre + 1er apellido + 1ra letra del 2do apellido (si no posee segundo apellido, se emplea la primera letra del primer apellido).

RBAC AES-256 Username Rule Idempotency

Escalamiento y Carga

Diseñado para operacion continua. La arquitectura desacoplada permite escalar el front-end y los workers de IA de forma independiente. Actualmente opera sobre DigitalOcean con Nginx, PHP-FPM, Redis y workers supervisados, manteniendo el mismo camino para aumentar capacidad.

Visibilidad y Logs

Contamos con un sistema de trazabilidad de respuestas de IA. Cada error de parsing o tiempo de espera se registra en logs detallados, permitiendo auditorías técnicas en tiempo real.

Modelo de Datos (Core)

La lógica reside en la relación operativa:
Interacción ➔ Transcripción ➔ Evaluación ➔ Feedback ➔ Insight.
Este flujo permite auditar lo que la IA detectó, lo que el monitor corrigió y cómo respondió el asesor desde web o app móvil. 

// Relación de Base de Datos
Interaccion hasMany Evaluaciones
Evaluacion hasOne AgentResponse
AgentResponse canOpen DisputeResolution

// Procesamiento
TranscribeAudioJob ➔ ScoreTranscriptJob ➔ InsightReport

Guía de Operación Estratégica

Carga y Evaluación

CRÍTICO

El monitor carga audios o textos, QA365 transcribe, evalúa con IA y deja una evaluación pendiente de revisión humana cuando corresponde.

Feedback & KPIs Móviles

KPI VIEW

El asesor puede aceptar o disputar su feedback en web o la APK. El sistema registra KPIs analíticos en la APK móvil para seguimiento de coaching: Tasa de Lectura (Visto), Tasa de Respuesta y Tasa de Aceptación.

Monitoreo Realtime y Ondas de Audio

REALTIME

Los dashboards se auto-actualizan cada 15 segundos en tiempo real. Los audios de evaluación integran un visor de ondas interactivo (Waveforms/Sentiment histogram) con soporte HTTP 206 Byte Range para búsquedas rápidas.

Insights y Reportes

TELEGRAM IA

El módulo de Insights consolida evaluaciones reales para generar análisis ejecutivo, hallazgos, riesgos y un guion dinámico con material de presentación para reuniones operativas.

Operación Escalable

"La escalabilidad se sostiene separando cargas: web para usuarios, workers para IA, colas Redis para picos de audio y límites controlados por proveedor."
